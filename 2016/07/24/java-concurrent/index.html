Java-concurrent

2016-07-24

java创建和销毁线程还是非常消耗资源的,以前我们都是通过new一个Thread对象出来,java1.5以后推荐使用线程池的方式创建线程

Executors类

java通过Executors类的静态方法创建线程池

  1. ExecutorService newFixedThreadPool(int nThreads):创建一个可重用的/具有固定数量的线程池
  2. ExecutorService newSingleThreadExecutor():创建一个线程池
  3. ExecutorService newCachedThreadPool():创建一个具有缓存功能的线程池，系统根据需要创建线程，这些线程将会被缓存在线程池中。
  4. ScheduledExecutorService newSingleThreadScheduledExecutor()
  5. ScheduledExecutorService newScheduledThreadPool(int corePoolSize)

ExecutorService

ExecutorService就代表一个线程池,程序只要传入一个Runnable或Callable的对象,该线程就会尽快的执行该任务.
主要有submit方法

Future主要用到的方法为get,如果线程还未执行完成,则阻塞等待执行完成后,再返回结果

ScheduleExecutorService

延迟,或者定时处理一些任务

附录A-Runnable和Callable的区别

Runnable只有一个run方法,没有返回值,Callable只有一个call方法,有返回值
