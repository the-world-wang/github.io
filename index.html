spring boot 日志管理

前言

Spring Boot在所有内部日志中使用Commons Logging，但是默认配置也提供了对常用日志的支持，如：Java Util Logging，Log4J, Log4J2和Logback。每种Logger都可以通过配置使用控制台或者文件输出日志内容。

日志级别

日志分为TRACE, DEBUG, INFO, WARN, ERROR, FATAL, OFF等级别，可以明确指定某个包的日志等级，如下：

logging.level.com.wang.util.version.ApiVersionRequestMappingHandlerMapping=error

文件输出

spring boot的日志默认输出到控制台，如果想输出到文件，可以指定logging.path和logging.file

logging.file=logs/test.log // 指明输出地址
logging.path=logs // 指明输出的路径
日志文件会在10Mb大小被截断，产生新的日志文件。
如产生test.log、test.log1，最新的在test.log中，也就是说之前的都丢了。

自定义日志配置

由于日志服务一般都在ApplicationContext创建前就初始化了，它并不是必须通过Spring的配置文件控制。因此通过系统属性和传统的Spring Boot外部配置文件依然可以很好的支持日志控制和管理。
根据不同的日志系统，你可以按如下规则组织配置文件名，就能被正确加载：

  • Logback：logback-spring.xml, logback-spring.groovy, logback.xml, logback.groovy
  • Log4j：log4j-spring.properties, log4j-spring.xml, log4j.properties, log4j.xml
  • Log4j2：log4j2-spring.xml, log4j2.xml
  • JDK (Java Util Logging)：logging.properties
    Spring Boot官方推荐优先使用带有-spring的文件名作为你的日志配置（如使用logback-spring.xml，而不是logback.xml）
