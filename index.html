mongo聚合操作

前言

mongo对自己的定义就是Aggregation Pipeline Operators，翻译成中文就是聚合管道操作符，这里的管道可以类比linux的管道，即将上一步的输出当做下一步的输入。

聚合框架

project投影

投影，类似于SQL中的select

  • 其中1表示投影，0表示不投影
  • 还可以重命名字段，用$title代表原字段

group

根据特定字段进行分组。如果选定好要分组的字段后，传给_id字段。

match

可以使用$gt,$in等，感觉放在前面相当于where，放在后面相当于having

lookup join

mongo 3.2后开始支持类join，相当于左外连接，但是这个不支持DBref

sort排序

limit/skip：分页

MapReduce

这个还不是很熟，有时间再写。

单一目的的聚合操作

  1. count：计算数量
  2. group：这个功能比$group还要多
  3. distinct：类似于SQL中的distinct

SQL映射表

SQL Mongo
WHERE $match
GROUP BY $group
HAVING $match
SELECT $project
ORDER BY $sort
LIMIT $limit
SUM() $sum
COUNT() $sum
join $lookup

spring boot 日志管理

前言

Spring Boot在所有内部日志中使用Commons Logging，但是默认配置也提供了对常用日志的支持，如：Java Util Logging，Log4J, Log4J2和Logback。每种Logger都可以通过配置使用控制台或者文件输出日志内容。

日志级别

日志分为TRACE, DEBUG, INFO, WARN, ERROR, FATAL, OFF等级别，可以明确指定某个包的日志等级，如下：

logging.level.com.wang.util.version.ApiVersionRequestMappingHandlerMapping=error

文件输出

spring boot的日志默认输出到控制台，如果想输出到文件，可以指定logging.path和logging.file

logging.file=logs/test.log // 指明输出地址
logging.path=logs // 指明输出的路径
日志文件会在10Mb大小被截断，产生新的日志文件。
如产生test.log、test.log1，最新的在test.log中，也就是说之前的都丢了。

自定义日志配置

由于日志服务一般都在ApplicationContext创建前就初始化了，它并不是必须通过Spring的配置文件控制。因此通过系统属性和传统的Spring Boot外部配置文件依然可以很好的支持日志控制和管理。
根据不同的日志系统，你可以按如下规则组织配置文件名，就能被正确加载：

  • Logback：logback-spring.xml, logback-spring.groovy, logback.xml, logback.groovy
  • Log4j：log4j-spring.properties, log4j-spring.xml, log4j.properties, log4j.xml
  • Log4j2：log4j2-spring.xml, log4j2.xml
  • JDK (Java Util Logging)：logging.properties
    Spring Boot官方推荐优先使用带有-spring的文件名作为你的日志配置（如使用logback-spring.xml，而不是logback.xml）

node小知识

此博客主要记录平常用node的一些小知识点。

dependencies 与 devDependencies 的区别

npm install安装npm包时，有2种方式可以把它们写进package.json文件中

  1. –save：会把依赖名称添加到dependencies下
  2. –save-dev：会把依赖名称添加到devDependencies下

npm run

npm会在package.json文件中寻找srcript区域。其实npm test和npm start是npm run test和npm run start的简写。事实上，你可以使用npm run来运行scripts里的任何条目。
使用npm run的方便之处在于，npm会自动把node_modules/.bin加入$PATH，这样你可以直接运行依赖程序和开发依赖程序，不用全局安装了。只要npm上的包提供命令行接口，你就可以直接使用它们。

node项目部署

可以利用supervisor或者forever、pm2管理进程
