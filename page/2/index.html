node小知识

此博客主要记录平常用node的一些小知识点。

dependencies 与 devDependencies 的区别

npm install安装npm包时，有2种方式可以把它们写进package.json文件中

  1. –save：会把依赖名称添加到dependencies下
  2. –save-dev：会把依赖名称添加到devDependencies下

npm run

npm会在package.json文件中寻找srcript区域。其实npm test和npm start是npm run test和npm run start的简写。事实上，你可以使用npm run来运行scripts里的任何条目。
使用npm run的方便之处在于，npm会自动把node_modules/.bin加入$PATH，这样你可以直接运行依赖程序和开发依赖程序，不用全局安装了。只要npm上的包提供命令行接口，你就可以直接使用它们。

node项目部署

可以利用supervisor或者forever、pm2管理进程

常用命令

命令 作用

语义化版本 2.0

摘要

  1. 主版本号：不兼容的API修改
  2. 次版本号：向下兼容的功能新增
  3. 修订号：向下兼容的bug修复
    如 1.2.3

问题

  1. 禁止在数字前方补零，如1.09.1
  2. 1.9.1 -> 1.10 -> 1.11.0
  3. 先行版本号：[0-9A-Za-z-]，常用的先行版本有：alpha，beta，rc，GA(稳定版)
  4. 版本编译信息：先加上一个加号再加上一连串以句点分隔的标识符号来修饰，如：1.0.0-alpha+001
